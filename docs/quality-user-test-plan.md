# Quality user test plan

这个文档用于模拟真实网络用户可能输入的内容，帮助判断 Gu Ren Said 的匹配质量是否够好。

重点不是只看“有没有结果”，而是看：

- 结果是否语义贴切
- 出处是否可靠
- 是否适合分享
- 是否适合学习原文
- 是否有明显误配

## 目标用户画像

### 1. 学生 / 作文用户

他们想把普通表达换成更高级的表达。

测试输入：

```text
我很难过
我很开心
我很想家
不要放弃
慢慢努力会变好
我想写一个关于坚持的句子
我想表达少年意气
我想表达孤独
我想表达释怀
我想表达不被理解
```

质量标准：

- 优先返回常见、可理解、适合作文引用的句子。
- 不要返回太冷门、太难懂、上下文不合适的句子。
- 解释要告诉用户为什么适合这个表达。

### 2. 小红书 / 朋友圈用户

他们想生成好看的分享卡片。

测试输入：

```text
你真好看
我爱你
我想你了
今天心情很好
今天破防了
这事包的
我真的会谢
太卷了想躺平
好朋友就是同频
一个人在外突然想家
```

质量标准：

- Top1 必须足够“好看”和适合传播。
- 卡片上句子不能太生僻。
- 解释不能太学术，要像给普通用户看的。

### 3. 亲子 / 老师用户

他们希望用网络热梗引导孩子学古诗文。

测试输入：

```text
孩子总说emo
孩子说666怎么换成古诗
孩子不想努力了怎么鼓励
孩子说想躺平
孩子说我真的会谢
孩子说这人太牛了
孩子说包的
```

质量标准：

- 结果要积极、可讲解。
- 不要返回价值观偏差太大的句子。
- 原文和出处要方便老师/家长继续讲。

### 4. 情绪表达用户

他们不是为了作文，而是表达情绪。

测试输入：

```text
我心态崩了
我有点丧
今天太累了
不想上班
想逃离一切
我被生活打败了
我释怀了
算了不想纠结了
```

质量标准：

- 不要把低落情绪误配成励志。
- 不要把释怀误配成悲伤。
- 情绪边界要清楚。

### 5. 爱情表达用户

这是最容易误配的场景，必须重点测。

测试输入：

```text
我爱你
我喜欢你
我想你
我想和你一辈子在一起
我只想和你在一起
我对你念念不忘
我暗恋你很久了
我想表白但不敢说
```

质量标准：

- “我爱你”不要返回单纯夸外貌的句子。
- “我想你”不要返回泛泛的爱情句，应该偏相思。
- “一辈子在一起”应该偏相守、承诺。
- “暗恋”应该偏心悦、未说出口。

## 当前最需要继续优化的问题

### 1. 召回仍然会偏粗

如果候选里没有合适句子，DeepSeek 只能在不够好的候选里排序。

下一步：

- 增加高频场景精选库。
- 对爱情、夸人、情绪、励志、思乡做专题精选。

### 2. 需要真实用户反馈入口

建议页面上每条结果增加：

```text
结果不准
出处有误
我有更好的句子
```

记录 query、返回结果、用户反馈。

### 3. 不能只看 Top1

有些 query 可能 Top1 一般，但 Top3 里有好结果。

下一步：

- 展示多条候选时，增加“更适合表白 / 更适合作文 / 更适合分享”的标签。

## 下一轮建议测试指标

发 V2EX / 公众号后，收集：

```text
1. 用户实际搜索词 Top 100
2. 用户指出的错误结果
3. 被用户认为“很惊艳”的结果
4. 用户希望新增的热梗
5. 用户是否愿意保存/分享卡片
```

## 判断是否继续投入

如果 7 天内出现以下信号，值得继续做：

```text
GitHub star > 50
V2EX 评论 > 20
有人主动提 issue / PR
用户在评论区贡献热梗或古诗
网站 UV > 500
有小红书/朋友圈二次传播截图
```

如果没有明显反馈，就把它定位成一个开源作品和内容案例，不继续重投入。
