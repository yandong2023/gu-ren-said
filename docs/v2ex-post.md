## 正文

最近看到一个挺有意思的视频。

大概是一个父亲教育女儿读书，说现在很多网络用语，其实古人早就讲过。

比如现在说：

```text
我 emo 了
这事包的
你真好看
我爱你
太卷了，想躺平
```

古人可能会说：

```text
抽刀断水水更流，举杯消愁愁更愁。
长风破浪会有时，直挂云帆济沧海。
巧笑倩兮，美目盼兮。
愿得一心人，白头不相离。
采菊东篱下，悠然见南山。
```

我觉得这个点挺适合做成一个小工具，于是做了一个：

```text
古人早就说过
https://gurensaid.com
```

GitHub：

```text
https://github.com/yandong2023/gu-ren-said
```

它不是普通的“文言文翻译器”。

我最开始也想过直接让大模型回答：

```text
“我爱你，古人怎么说？”
```

但这样很容易有一个问题：

大模型可能会编一句很像古文的话，然后再编一个看起来很像真的出处。

所以现在的思路是：

```text
用户输入现代话 / 网络热梗
        ↓
理解语义，生成检索线索
        ↓
去古诗文数据库里找真实存在的句子
        ↓
只从真实候选里排序和解释
        ↓
生成可分享的手机卡片
```

也就是说，最终展示的句子必须来自数据库，带作者、朝代、篇名和原文。

目前做了这些功能：

```text
1. 输入现代话，反查意思相近的古诗文
2. 展示原句、作者、篇名、解释和原文
3. 生成 3:4 分享卡片
4. 微信里不能直接分享图片时，支持生成图后长按保存
5. 接入了 DeepSeek 做语义理解和候选排序，但不让它直接编古文
6. 开源了代码和数据导入逻辑
```

几个例子：

```text
我爱你
→ 愿得一心人，白头不相离。
→ 汉·卓文君《白头吟》

你真好看
→ 巧笑倩兮，美目盼兮。
→ 先秦·佚名《诗经·硕人》

我 emo 了
→ 抽刀断水水更流，举杯消愁愁更愁。
→ 唐·李白《宣州谢朓楼饯别校书叔云》

这事包的
→ 长风破浪会有时，直挂云帆济沧海。
→ 唐·李白《行路难·其一》
```

目前的问题也挺明显：

```text
1. 有些表达匹配得不错，有些还是会不够贴切
2. 古诗文不是简单语义检索，很多句子有上下文，不适合乱用
3. 分享体验在微信内置浏览器里比较受限制
4. 高频表达需要人工精选，不然大库召回会比较粗
```

我现在比较想验证几个问题：

```text
1. 这个东西大家觉得有意思吗？
2. 它更适合做成开源玩具，还是可以继续做成小产品？
3. 这种“现代话 → 古文真实出处”的交互，大家觉得应该怎么优化？
4. 有没有你觉得特别适合加入的网络热梗？
5. 如果你搜出来的结果不准，欢迎直接贴出来，我想继续调一下质量。
```
